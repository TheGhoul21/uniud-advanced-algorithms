\section{Definition}

The exact pattern matching problem is defined as follows:
\begin{problem}[Exact pattern matching]
  Given an alphabet $\Sigma$, and two strings $T$, $P$ $\in \Sigma^*$, for which $|T| = n$ and $|P| = m$, find all
  occurrences of $P$ in $T$.
\end{problem}

\begin{example}
  Let $\Sigma = \{a, b\}$, $T = \texttt{ababbab}$ and $P = \texttt{abb}$. The occurrences of $P$ in $T$ are at
  positions $2$ and $4$.
\end{example}

What the Figure \ref{fig:naive-matching} illustrates is one basic naive approach. In this approach we nest two loops: the outer loop iterates over all possible "first positions" $s$ in $T$ where the pattern $P$ could potentially match, and the inner loop checks character by character if $P$ matches $T$ starting from position $s$. 
For example the first row of the figure corresponds to starting the search from index $s=0$ of $T$. Given $s$ we then loop one by one each character of $P$, moving along $T$ as well, and check if all the characters match. In this case we find a mismatch at the third character of $P$ (which is 'b', whereas the corresponding character in $T$ is 'a'). When this happens, we break this inner loop and move to the next shift $s=1$. 
If the lenght of the pattern was $m$ and the length of the text was $n$, the first outer iterations could take up to $O(m)$ time, and we could have up to $O(n)$ such iterations, leading to a worst-case time complexity of $O(nm)$ for this naive approach.

In figure \ref{fig:naive-worst-case} we see an example of a worst-case input for the naive algorithm. 
In this case the text $T$ consists of $n$ 'a' characters, and the pattern $P$ consists of $m-1$ 'a' characters followed by a 'b'.
There are $n-m+1$ possible shifts $s$ where the pattern could potentially match, and for each of these shifts the algorithm will successfully match the first $m-1$ characters of $P$. In the example of the figure we have $n=7$ and $m=4$, so there are $7-4+1=4$ shifts to check, and for each shift we make $m$ checks, of which $m-1=3$ are successful matches before we encounter the mismatch at the last character of $P$. 
This results in a total of $(n-m+1)(m-1)$ comparisons, which is $\Theta(nm)$ in the worst case.

This naive approach basically starts over from scratch at each shift, without leveraging precious information from previous comparisons. 

\section{Knuth-Morris-Pratt (KMP) algorithm}

What if we could somehow "remember" the information from a previous comparison, and use it to skip some of the checks at the next shift?
Let's introduce the intuition behind the Knuth-Morris-Pratt (KMP) algorithm, with an example.

\begin{example}
  \label{ex:kmp-intro}
  Let $P = \texttt{abcdeabdef}$ and suppose we are running the naive algorithm at shift $s=2$
  over some text $T$ whose characters at positions $2$--$8$ are \texttt{abcdeab}.
  We match $P[0\ldots6]$ successfully, then hit a mismatch at $P[7] = \texttt{d}$ vs $T[9] = \texttt{x}$.
\end{example}

As shown in Figure~\ref{fig:kmp-shift}, at this point the naive algorithm would discard everything
and try shift $s=3$. But notice that the matched portion $\texttt{abcdeab}$ starts \emph{and} ends
with $\alpha = \texttt{ab}$: the prefix $P[0\ldots1]$ and the suffix $P[5\ldots6]$ of the matched
region are identical. This means that when we slide $P$ forward, the $\alpha$ that was at the end
of the match is now perfectly aligned with the $\alpha$ at the start of $P$ --- we can jump
directly to shift $s=7$ and resume comparing from $P[2]$, skipping five shifts entirely.

\aside{Important: when we say suffix we mean suffix of the \textbf{matched} portion of $P$, not necessarily a suffix of the whole pattern. In the example, $\alpha$ is a suffix of $P[0\ldots6]$ but not of $P$ itself.}
Basically what the example shows is that if we take the longest prefix of $P$ that is also a suffix of the \textbf{matched} portion of $P$, we can shift the pattern so that this prefix is now aligned with the suffix --- and we can resume the search from the character right after this prefix, since we know it must match (otherwise we would have had a mismatch earlier). In other words we have saved some works (namely, three shift indexes) by leveraging the information from the previous comparisons, instead of starting over from scratch, from the very next shift.

But is this correct? Or do we miss some potential matches by jumping directly to $s=7$?

Well, it turns out that this is indeed correct, and we won't miss any matches by doing this. The KMP algorithm is based on this very idea, and it guarantees that we will find all occurrences of $P$ in $T$ without missing any, while also achieving a much better time complexity than the naive approach.

\subsection{Computing the failure function}

\aside{A \emph{proper} prefix/suffix excludes the whole string itself.}
To exploit this idea, we need to precompute for every prefix $P[0\ldots j]$ its
\emph{longest proper prefix that is also a suffix}, written $\pi[j]$.
This table is known as the \textbf{failure function} (or prefix function) of $P$.

\begin{definition}[Failure function]
  Let $P$ be a pattern of length $m$. The failure function $\pi : \{0,\ldots,m-1\} \to \{0,\ldots,m-1\}$
  is defined as
  \[
    \pi[j] \;=\; \max\bigl\{\, k < j+1 \mid P[0\ldots k-1] = P[j-k+1\ldots j] \,\bigr\},
  \]
  i.e.\ the length of the longest proper prefix of $P[0\ldots j]$ that is also a suffix of $P[0\ldots j]$.
\end{definition}

The key observation is that $\pi[j]$ can be computed recursively from $\pi[j-1]$,
giving an efficient $\Oh{m}$ preprocessing algorithm. There are two cases:

\begin{enumerate}
  \item \textbf{Extension is possible.} If $P[\pi[j-1]] = P[j]$, then the longest
        prefix-suffix of $P[0\ldots j]$ is simply one character longer than that of
        $P[0\ldots j-1]$, so $\pi[j] = \pi[j-1] + 1$.

        \textit{Example:} with $P = \texttt{abcdeabdef}$ at $j=6$, we have $\pi[5]=1$
        (prefix-suffix of \texttt{abcdea} is \texttt{a}). We check $P[1] = \texttt{b}$
        against $P[6] = \texttt{b}$: they match, so $\pi[6] = 2$.

  \item \textbf{Extension is not possible.} If $P[\pi[j-1]] \neq P[j]$, we cannot
        extend the current prefix-suffix. We then ask: what is the longest
        prefix-suffix of the prefix-suffix itself? In other words, we fall back to
        $\pi[\pi[j-1]-1]$ and try to extend that instead. We repeat this process
        until either a match is found or we reach length $0$, in which case $\pi[j] = 0$.

        \textit{Example:} still with $P = \texttt{abcdeabdef}$ at $j=7$, we have
        $\pi[6]=2$ (prefix-suffix of \texttt{abcdeab} is \texttt{ab}). We check
        $P[2] = \texttt{c}$ against $P[7] = \texttt{d}$: no match. We fall back to
        $\pi[1] = 0$, so we try extending a prefix-suffix of length $0$: we check
        $P[0] = \texttt{a}$ against $P[7] = \texttt{d}$, still no match. We have
        reached length $0$, so $\pi[7] = 0$.
\end{enumerate}

  We then compare $P[k] = P[1] = \texttt{b}$ against $P[j] = P[6] = \texttt{b}$
  (\textcolor{orange!70!black}{orange}): they match, so we extend and set $\pi[6] = 2$.
}
\label{fig:failure-function}
\end{figure}

The resulting algorithm is the following:

\begin{algorithm}[H]
  \caption{KMP failure function}
  \KwIn{Pattern $P[0\ldots m-1]$}
  \KwOut{Failure function $\pi[0\ldots m-1]$}
  $\pi[0] \gets 0$ \;
  $k \gets 0$ \tcp*{length of current candidate prefix-suffix}
  \For{$j \gets 1$ \KwTo $m-1$}{
    \While{$k > 0$ \textbf{and} $P[k] \neq P[j]$}{
      $k \gets \pi[k-1]$ \tcp*{fall back}
    }
    \If{$P[k] = P[j]$}{
      $k \gets k + 1$ \;
    }
    $\pi[j] \gets k$ \;
  }
  \Return $\pi$
\end{algorithm}

\begin{remark}
  The ``fall back to the prefix-suffix of the prefix-suffix'' is the recursive
  heart of KMP: the same shift logic used during the search phase is also used
  during preprocessing.
\end{remark}

\begin{exercise}
  Design an algorithm that, given a pattern $P$ of length $m$, computes
  $\pi[i]$ for every $i \in \{0, \ldots, m-1\}$ in $\Oh{m}$ time.

  \medskip
  \textit{Hints:}
  \begin{enumerate}
    \item Try to express $\pi[i]$ in terms of $\pi[i-1]$.
          What is the relationship between the longest prefix-suffix of
          $P[0\ldots i]$ and that of $P[0\ldots i-1]$?
    \item There are two cases depending on whether $P[\pi[i-1]]$ equals
          $P[i]$ or not. What can you do in each case?
    \item For the harder case, think recursively: if you cannot extend
          a prefix-suffix of length $k$, which shorter candidate should
          you try next?
  \end{enumerate}
\end{exercise}
